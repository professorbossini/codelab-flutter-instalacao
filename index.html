Flutter é um arcabouço que viabiliza o desenvolvimento multiplataforma. Utilizando a linguagem de programação Dart, podemos desenvolver aplicações para

  • Android
  • iOS
  • Linux
  • macOS
  • Web
  • Windows

Neste codelab, veremos como fazer a instalação do ambiente para desenvolvimento na plataforma Windows.

O primeiro passo é visitar a página oficial do framework para obtê-lo.

Flutter - Home Page

A seguir, clique em Get Started.

Agora podemos escolher a plataforma em que desejamos fazer a instalação. Clique em Windows.

Observe que os requisitos mínimos são exibidos. Além disso, duas dicas muito importantes também são exibidas.

Cuidado. Não instale o Flutter em diretórios que possuam caracteres especiais ou espaços.

Cuidado. Não instale o Flutter em diretórios envolvidos com o mecanismo de permissões do sistema operacional, como a pasta Arquivos de Programas do Windows.

Clique para fazer o download do arquivo. Isso pode demorar um pouco já que o arquivo é "grande".

Observe que o arquivo obtido tem a extensão zip. Você provavelmente poderá encontrá-lo na pasta de Downloads padrão de seu sistema operacional ou naquela que tiver configurado em seu navegador. Lembre-se que é recomendável manter a instalação do Flutter em um diretório não envolvido com o mecanismo de permissões do sistema operacional. Uma boa opção é o diretório do usuário do sistema operacional com que você está logado no momento.

No Windows Explorer, comece clicando em Meu Computador para encontrar o Disco Local C.

Depois de clicar duas vezes em Disco Local (C:), clique duas vezes na pasta Usuários.

Nota. Pode ser que os nomes das pastas sejam apresentados em inglês.

Agora, clique duas vezes na pasta que leva o nome do usuário do sistema operacional logado no momento. Neste exemplo, o usuário se chama rodri.

Agora, encontre uma pasta chamada Documents - ou Documentos - e clique duas vezes sobre ela.

Clique com o direito no vazio e escolha Novo >> Pasta. Observe.

Escolha um nome para a pasta: flutter pode ser uma boa ideia.

Clique duas vezes sobre a pasta que você acabou de criar. Agora, você precisa recortar o arquivo de instalação do Flutter e colar ele na sua pasta. Ele está na sua pasta de Downloads, lembra?

Clique com o direito no arquivo e escolha Extrair Tudo....

Na tela a seguir, lembre-se de escolher a pasta atual como destino e clique em Extrair.

Esse processo pode demorar alguns minutos. Quando ele terminar, você pode excluir o arquivo zip, assim economiza espaço no seu disco. Clique com o direito sobre ele e escolha Excluir.

Os utilitários de que precisaremos se encontram na pasta bin do diretório de instalação. Para que eles possam ser executados a partir da linha de comando, precisamos associar este diretório à variável de ambiente chamada Path do usuário.

Comece copiando o endereço completo da pasta bin.

Agora, clique no botão iniciar do Windows e busque por env. Clique na opção Editar as variáveis de ambiente para a sua conta.

Encontre a variável Path do seu usuário, clique uma vez sobre ela e clique em Editar.

Clique em Novo, cole o endereço da pasta bin que você havia copiado e clique em OK.

Agora podemos verificar se a instalação foi feita corretamente.

Cuidado. Para realizar os testes a seguir, certifique-se de fechar quaisquer terminais (powershell, cmd etc) que estejam em funcionamento. As configurações envolvendo a variável Path não terão efeito em terminais que tenham sido abertos antes de elas terem sido realizadas.

Para realizar os testes, use Windows+R no teclado e digite cmd. Clique em OK.

Use

Where flutter dart

na esperança de obter o seguinte resultado, que indica que os utilitários flutter e dart podem ser executados na linha de comando.

Lembre-se que o Flutter viabiliza o desenvolvimento multiplataforma. Cada plataforma possui algumas dependências que precisam estar instaladas adequadamente para que o desenvolvimento possa ser realizado. Use

flutter doctor

para verificar se você possui dependências ainda não completamente instaladas. Veja um resultado possível.

Observe que há dois problemas a serem resolvidos. No que diz respeito ao Android, há uma pendência quanto ao aceite de termos de uso. Observe que a própria mensagem já inclui o comando a ser executado para supostamente resolver o problema. Vá em frente e execute

flutter doctor --android-licenses

A mensagem que aparece lhe pergunta se você deseja visualizar os termos que ainda não foram aceitos.

Digite y e aperte Enter.

Você visualizará os termos de uso de algum produto que eventualmente vá utilizar e que ainda não tenha aceito. Um exemplo é o android-googletv. Lembre-se que em sua máquina podem existir variações.

Digite y e aperte Enter novamente e repita esses passos (ler os termos e aceitar) para cada novo item que aparecer. Veja o resultado esperado após aceitar todos os termos necessários.

Depois disso, execute

flutter doctor

mais uma vez. Veja o resultado.

Neste exemplo, ainda há uma pendência: o Visual Studio não está instalado, o que é necessário caso queiramos desenvolver tendo como alvo o ambiente Windows. Se for o seu caso, siga o link da mensagem e faça as instalações.

Cuidado. O Visual Studio Code e o Visual Studio são ferramentas diferentes. É comum confundir.

Cuidado. Embora tenhamos como objetivo desenvolver com Flutter no ambiente Windows, isso não quer dizer que teremos esse ambiente como alvo. Nada impede, por exemplo, que desenvolvamos no Windows tendo como alvo o Android e o Linux. Ou seja, o aplicativo desenvolvido é que será executado nestes dois ambientes.

Parabéns! Você acaba de instalar o Flutter e está pronto para começar a aprender a programar utilizando a linguagem Dart, o próprio framework Flutter e a desenvolver aplicações para múltiplas plataformas.
